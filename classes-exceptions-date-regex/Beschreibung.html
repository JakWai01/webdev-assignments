D1: class für einen Counter Button

  • Erstellen sie in der Vorlage die Klasse MyCounter die von HTMLElement erbt. Fügen sie folgendes zur Klasse hinzu:
    • Eine Property count mit dem Standardwert 0.
    • Die Methode render() die this.innerText auf "count: 123" setzt. Wobei statt "123" natürlich der aktuelle Wert der count-Property eingesetzt werden soll.
    • Einen Konstruktor der den Konstruktor der Elternklasse aufruft.
    • Zudem soll der Konstruktor der onclick-Property eine Arrow-Function zuweisen die count um 1 erhöht und render() aufruft.
    • Testen sie was passiert wenn sie auf das my-counter-Element klicken.
  • Fügen sie die Methode connectedCallback() zur Klasse hinzu. Diese nimmt keine Parameter engtegen und soll render() aufrufen. Diese Callback wird vom Browser automatisch aufgerufen sobald ein my-counter-Element in einem Dokument verwendet wird.
  • Schreiben sie die render()-Methode um so dass folgender Text ausgegeben wird (123 ist wie oben nur ein Platzhalter):
    • normale ausgabe: "😐 123"
    • ab counter 2: "🤔 123"
    • ab counter 5: "🙂 123"
    • ab counter 10: "😺 123" oder "🐶 123"

    Sie können die Emoticons einfach als Text markieren, kopieren und in JavaScript-Strings einfügen.

D2: Throw, Try - Catch - Finally

Validieren Sie das InputFeld bei Klick auf den Button.

  • Es soll ein Error geworfen werden throw wenn:
    • der String leer ist
    • die Eingabe keine Zahl ist isNaN
    • die Zahl kleiner als 5 oder größer als 10 ist
    • Für diese Validierung muss der input erst mit parseInt() konvertiert werden
  • Nutzen Sie try-catch-finally um:
    • Bei einem Fehler die geworfene Fehlermeldung auszugeben: Input is {error}
    • Bei einer validen Eingabe Input is {value} auszugeben.
    • Nach jeder Aktion das Input-Feld wieder zu leeren (egal ob die Eingabe valide oder invalide war).

D3: Monatskalender

  • Erstellen sie eine Date-Variable die den aktuellen Tag repräsentiert.
  • Benutzen sie diese Variable um ein neues Date für den ersten Tag des Monats zu berechnen.
  • Verwenden sie eine Schlefe um über alle Tage des aktuellen Monats zu laufen (vom ersten bis zum letzten Tag). Für jeden Tag soll `<dt>${day.toLocaleDateString()}</dt>` zum innerHTML des dl-Elements hinzugefügt werden.
  • Erweitern sie die Schleife und suchen sie für jeden Tag nach passenden Objekten im events-Array. Jedes gefunde Event soll als `<dd>${event.name}</dd>` nach dem <dt> des Tages ausgegeben werden.

D4: RegExp

  • Stellen sie sicher dass in das Eingabefeld der Vorlage (input-Element) nur ein HdM-Kürzel eingegeben werden kann. Tragen sie dazu in das pattern-Attribut einen passenden regulären Ausdruck ein (ohne "/" außen rum).

    Falls die Eingabe nicht zum regulären Ausdruck passt wird der Text durch CSS rot angezeigt.

    Sie können Tools wie regex101.com verwenden um den regulären Ausdruck zu schreiben und zu testen.

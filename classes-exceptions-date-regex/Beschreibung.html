D1: class fr einen Counter Button

  • Erstellen sie in der Vorlage die Klasse MyCounter die von HTMLElement erbt. Fgen sie folgendes zur Klasse hinzu:
    • Eine Property count mit dem Standardwert 0.
    • Die Methode render() die this.innerText auf "count: 123" setzt. Wobei statt "123" natrlich der aktuelle Wert der count-Property eingesetzt werden soll.
    • Einen Konstruktor der den Konstruktor der Elternklasse aufruft.
    • Zudem soll der Konstruktor der onclick-Property eine Arrow-Function zuweisen die count um 1 erhht und render() aufruft.
    • Testen sie was passiert wenn sie auf das my-counter-Element klicken.
  • Fgen sie die Methode connectedCallback() zur Klasse hinzu. Diese nimmt keine Parameter engtegen und soll render() aufrufen. Diese Callback wird vom Browser automatisch aufgerufen sobald ein my-counter-Element in einem Dokument verwendet wird.
  • Schreiben sie die render()-Methode um so dass folgender Text ausgegeben wird (123 ist wie oben nur ein Platzhalter):
    • normale ausgabe: " 123"
    • ab counter 2: " 123"
    • ab counter 5: " 123"
    • ab counter 10: " 123" oder " 123"

D2: Throw, Try - Catch - Finally

Validieren Sie das InputFeld bei Klick auf den Button.

  • Es soll ein Error geworfen werden throw wenn:
    • der String leer ist
    • die Eingabe keine Zahl ist isNaN
    • die Zahl kleiner als 5 oder grer als 10 ist
    • Fr diese Validierung muss der input erst mit parseInt() konvertiert werden
  • Nutzen Sie try-catch-finally um:
    • Bei einem Fehler die geworfene Fehlermeldung auszugeben: Input is {error}
    • Bei einer validen Eingabe Input is {value} auszugeben.
    • Nach jeder Aktion das Input-Feld wieder zu leeren (egal ob die Eingabe valide oder invalide war).
